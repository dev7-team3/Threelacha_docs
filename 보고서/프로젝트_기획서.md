## 1. **프로젝트 소개**

> 농축수산물 공공데이터를 수집·분석·시각화하여, 
소비자의 합리적인 구매 의사결정 돕는 대시보드 구축 프로젝트
> 

## 2. **문제 정의 및 기획 의도**

### 2-1. 추진 배경 및 문제 정의

- **고물가 시대, 식탁 물가 부담 가중**
    
    **:** 고물가와 외식비 상승으로 인해 '집밥' 수요가 늘고 있으나, 농축수산물의 가격 급등락으로 인해 장바구니 물가 부담은 여전히 높음.
    
- **가격 정보의 비대칭성**
    
    **:** 농축수산물 가격은 지역·유통채널·시점에 따라 변동성이 매우 크지만, 기존 공공 서비스는 단순 가격 조회에 그쳐 종합적인 비교가 어려움.
    
- 친환경 농산물의 경우 일반 농산물 대비 높은 정보 장벽
    
    **:** 일반 농산물 가격 정보에 비해 친환경 식재료의 가격 정보는 접근 경로가 제한적이고 파편화되어 있음.
    
- **구매 의사결정의 어려움**
    
    **:** 소비자는 "언제, 어디서, 무엇을" 사야 가장 합리적인지 판단하기 어려우며, 가격 급등 시 대안(대체재)을 찾는 데 정보 탐색 비용이 발생함.
    

### 2-2. 기획 의도 및 핵심 솔루션

- **핵심 솔루션**
    - **최적의 구매 시점 포착 (Timing)**
        
        > "지금 사는 것이 쌀까? 조금 더 기다릴까?"
        > 
        - **기능:** 과거 데이터 기반의 가격 등락 추이 시각화 및 변동성 지표 제공.
        - **가치:** 소비자가 일시적인 급등락에 휘둘리지 않고, 가격 안정기를 파악하여 구매 시점을 조절할 수 있도록 지원.
    - **채널별 물가 지도 & 최저가 탐색 (Where)**
        
        > "대형마트와 전통시장, 어디가 더 저렴할까?"
        > 
        - **기능:** 유통채널(대형마트, 전통시장, 슈퍼마켓 등)별 가격 비교 및 가격 분포 시각화.
        - **가치:** 품목별로 경쟁력 있는 유통채널 정보를 제공하여, 소비자가 고정관념에서 벗어나 채널을 교차 선택(Mix & Match)하도록 유도.
    - **가치 소비의 합리화 (Eco-friendly)**
        
        > "친환경은 무조건 비싸다는 편견 깨기"
        > 
        - **기능:** 일반 농산물과 친환경 농산물의 가격 차이(Premium) 및 변동 추이 비교 분석.
        - **가치:** 가격 격차가 좁혀지는 시점을 포착해 제시함으로써, 막연한 가격 부담감을 낮추고 친환경 소비 접근성 확대.
- **개발 고려 확장 솔루션**
    - **'진짜 가격' 비교를 위한 표준화 (Unit Price)**
        
        > "한 묶음과 100g, 도대체 무엇이 더 이득일까?"
        > 
        - **기능:** 판매 단위가 제각각인 농산물 가격을 '표준 단위(예: 100g, 1개)'로 환산하여 제공.
        - **가치:** 포장 단위에 현혹되지 않고 직관적인 가격 비교를 가능하게 하여 실질적인 비용 절감 유도.
    - **유연한 소비를 위한 대체재 제안 (Alternative)**
        
        > "상추가 너무 비싸면 깻잎은 어떨까?"
        > 
        - **기능:** 특정 품목 가격 급등 시, 가격 상관관계가 높거나 용도가 유사한 대체 품목의 가격 흐름 동시 제시.
        - **가치:** 가격 급등 품목을 고집하지 않고 저렴한 대체재로 유도하여 가계 부담 완화 및 식재료 선택의 폭 확장.

### 3-2. ETL/ELT 과정에서 수행할 작업

- 결측치 및 이상치 처리
- 칼럼 정규화 (가격 : String → Numeric / 날짜 : String → Date)
- 지역코드·부류코드·품목코드 등 코드 기반 칼럼 한글 정보 확장 (디멘전 테이블 조인)
- 날짜 데이터 기반 요일 정보 추가
- 단위 (kg, g, 개, 마리, 포기, 구 )  별 가격 환산 정보 칼럼 생성
- 수집 데이터 포맷 변환 저장 (xml/json → parquet)

### 3-3. 데이터 모델링(DBT) 방향

- 일간/주간/월간/연간 가격 변동률 마트
- 친환경 vs 일반 농축수산물 가격 비교 마트
- 유통채널(전통시장/마트/온라인)별 가격 마트
- 품목 분류별(예: 채소류, 축산물, 수산물) 가격 지수(Price Index) 마트
- 전년 동월, 전년 동기 대비 가격 비교 및 계절성 요인 분석 마트
- 품목 간 가격 상관관계 분석 마트

### 3-4. 대시보드 지표/인사이트

- **변동성 포착:** "금일 가격이 **가장 많이 오른 / 내린** 농축수산물 TOP 5"
- **가치 소비:** "친환경 농산물 중 **일반 상품 대비 가격 격차가 가장 적은** (가장 합리적인) 품목"
- **구매 장소:** "**같은 품목**, 전통시장 vs 대형마트 vs 온라인 가격 차이 및 **현재 최저가 채널**"
- **장기 전략:** "최근 n 개월 간 **가격 변동성이 가장 높은 / 낮은** 품목 (위험/안정 품목 인지)"
- **위기 대응:** "**가격 급등 품목의 대체 가능 품목 추천** (대체재 기반 소비 비용 절감)"

    - 팀원 간 이해도 정렬 및 방향성 재확인

## 7. **일정 계획**

- **Week 1 : 프로젝트 정의 및 설계**
    - 프로젝트 주제 확정 및 수집 대상 데이터 분석
    - GitHub 디렉토리 구조, 코드 컨벤션, Git-flow 등 **팀 개발 규칙 수립**
    - AWS 기반 데이터 파이프라인 아키텍처 초안 및 다이어그램 설계
- **Week 2 : 로컬 개발 환경 구성 및 데이터 수집**
    - AWS 파이프라인 구조를 기준으로 로컬 개발 환경 사전 구성
    - Airflow, MinIO, dbt, Trino 기반 데이터 수집·정제 파이프라인 구현
    - 팀원 간 환경 설정 및 공통 구성 요소 통일
    - 대시보드 초기 프로토타입 설계 및 주요 지표 정의
- **Week 3 : 클라우드(AWS) 환경 전환 + CI/CD 적용**
    - 로컬 파이프라인을 AWS 환경(S3, Airflow, dbt, Athena)으로 전환
    - Airflow 기반 배치 스케줄링 운영 환경 구성
    - GitHub Actions를 활용한 CI/CD 파이프라인 적용
    - 코드 변경 시 자동 배포 및 테스트 흐름 구축
- **Week 4 : 기능 고도화 및 안정화**
    - 데이터 모델 및 대시보드 기능 보완
    - 성능 개선 및 예외 처리 강화
    - 네트워크 보안 설정 및 서비스 자동 기동 환경 구성
    - 운영 관점에서의 파이프라인 안정성 확보
- **Week 5 : 결과 정리 및 발표 준비**
    - 최종 발표 자료(PPT) 작성
    - 결과 보고서(고용노동부 제출용) 작성
    - 최종 발표 시연 영상 제작
    - 클라우드 자원 정리·백업
